Paramorama - Getting Started

Input data

Paramorama reads input in plain text tab separated value files (TSV). For our purposes, each column in a TSV file represents a different parameter. Each row represents a record/tuple consisting of a value for each parameter. The image(s) associated with each record/tuple are referenced in one or more columns that include paths to where the images are stored on disk. For example:

First parameter Second parameter Output image
0.1 0.2 C:\IMages\output1.jpg
0.1 0.2 C:\IMages\output2.jpg

There are a few things to note. Paramorama automatically identifies those columns of which the values vary, so even if it turns out that a certain parameter stays constant it is not necessary to intercept this and prevent it from printing to file. Paramorama automatically detects columns of which the entries end with the extension ".jpg/.jpeg", so you do not have to encode anything special in the column headers to flag images. Paramorama currently reads jpeg images - due to good support in Java and various libraries I use - so it may be necessary to convert to this format at some point. There is an assumption that all images are the same size. I have not actually tested what happens when this is not the case, but I do not expect this to be an issue. The prefixes "tag_", is currently reserved for encoding user-specific tagging information and is best avoided for column headers.

I have implemented a sampling plug-in for CellProfiler to generate data for Paramorama. It currently ships with the developer's version of CellProfiler.

Using Paramorama

  1. To load data, select File > Open data file from the the menu bar.
